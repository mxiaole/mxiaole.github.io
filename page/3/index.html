数据结构和算法

算法专题

学习算法，其实是学习算法的基本思想，了解了算法的基本思想之后，用代码实现其实是很容易的。学习算法的前提是了解常用的数据结构：数组、栈、队列、堆、链表等。在本专题中默认大家已经掌握常用的数据结构，不会专门来讲解这些数据结构。

数据结构这一学科正是以“数据”这一信息的表现形式为研究对象，旨在建立支持高效算法的数据信息处理策略、技巧与方法。要做到根据实际应用需求自如地设计、实现和选用适当的数据结构，必须首先对算法设计的技巧以及相应数据结构的特性了然于心。

算法是依托于数据结构而存在的。

数据的逻辑结构，所谓的数据的逻辑结构是指，数据元素及其之间的关系，主要包括如下的逻辑结构：

  • 集合: 数据中的元素除了属于同一个集合外，没有其他的关系
  • 线性结构: 数据中的元素存在一对一的相互关系
  • 树形结构: 数据中的元素存在一对多的关系
  • 图形结构:

搞清楚研究对象是什么？研究对象中的数据元素是什么？数据元素之间有什么关系？

1 排序算法

1.1 选择排序

O(n^2)

选择排序的基本思想就是：
1.从给定的数组中找到最小的放在1号位置
2.从剩下的元素中找出最小的位置，放在当前数组的1号位置
3.重复上述过程，直到最后数组中的元素为0

1.2 插入排序

O(n^2)

基本思想：
1.将数据分成两部分，一部分有序，一部分无序
2.每次从无序的部分取出一个数，插入到有序的数组中

分析：如果待排序的数组是基本有序的，那么排序的效率是非常的高的。如果待排序的数组是有序的，那么排序的时间复杂度就会是O(n)

etcd源码分析-环境搭建

1. 编译

  • 源码下载：git clone https://github.com/etcd-io/etcd.git

  • 源码的编译：cd etcd源码目录，执行make，直接对项目进行编译。ectd的编译使用Makefile和shell的方式完成。通过执行make命令，调用shell脚本来完成编译的工作

makefile文件：

build文件：

go build命令参数：

  • 如果一个项目没有依赖GOPATH下的任何的包，那么直接go build命令就可以实现编译。此时会自动搜索当前目录下的源码。

  • 可以给go build指定编译的源码：go buid main.go file1.go file2.go，此时会默认使用第一个文件名作为编译输出的可执行文件名

  • go build -o main package，包名是相对于GOPATH下的src目录开始的，-o 指定输出的可执行文件的名字

  • 其他的参数：

    • -v : 编译时显示包名
    • -ldflags：指定传递给链接器的参数。
    • -gcflags: 指定传递给编译器的参数
    • -x: 查看并执行编译命令
    • -a: 强制重新编译所有依赖包
    • -installsuffix： 为了使当前的输出目录与默认的编译输出目录分离，可以使用这个标记。此标记的值会作为结果文件的父目录名称的后缀。其实，如果使用了-race标记，这个标记会被自动追加且其值会为race。如果我们同时使用了-race标记和-installsuffix，那么在-installsuffix标记的值的后面会再被追加_race，并以此来作为实际使用的后缀。
