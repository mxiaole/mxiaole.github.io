maven使用

Posted on 2020-03-05 In 工具

project：要么是一个可执行的程序，要么是提供给其他project使用的包或者模块。

project的生命周期：

  • 创建
  • 编码
  • 编译
  • 测试
  • 打包
  • 部署

maven通过插件来对project的生命周期进行管理。

maven安装：将中央仓库改为阿里云镜像conf/setting.xml文件中添加

1 简介

使用一个标准的目录结构，然后使用一个pom.xml文件放在工程的根目录下，可以实现：

  • 对当前的工程根据 pom.xml 中描述的打包配置创建jar包或者war包，然后可以直接发布
  • 在编译的时候，会自动的下载项目所依赖的jar包

1.1 maven项目的目录结构

  • {baseDir}/src/main/java 源代码，sources
  • {baseDir}/src/main/resources 资源, resources
  • {baseDir}/src/test/java 测试文件，Tests
  • {baseDir}/target 生成的目标

2.2 pom文件

在pom文件中表示出：

  • 工程要生成的包名：groupId:artifactId:version三个部分组成。groupId为公司名，artifactId为项目名称，version为项目的版本
  • 工程的依赖jar包

2.3 构建maven项目

使用maven生成一个jar包的过程：

  • 资源拷贝
  • 源代码编译
  • 生成jar包
  • 将生成的jar包发布到本地或者远程仓库中

2.4 仓库

maven仓库有三种类型：

  • 本地：本地机器上的一个文件夹，当你运行一次 Maven 构建，Maven 会自动下载所有依赖的 jar 文件到本地仓库中。本地仓库的默认路径是%HOME/.m2
  • 中央：由 Maven 社区提供的仓库
  • 远程：开发人员自己定制仓库，包含了所需要的代码库或者其他工程中用到的 jar 文件

当我们执行 Maven 构建命令时，Maven 开始按照以下顺序查找依赖的库：

  • 步骤 1 － 在本地仓库中搜索，如果找不到，执行步骤 2，如果找到了则执行其他操作。
  • 步骤 2 － 在中央仓库中搜索，如果找不到，并且有一个或多个远程仓库已经设置，则执行步骤 4，如果找到了则下载到本地仓库中已被将来引用。
  • 步骤 3 － 如果远程仓库没有被设置，Maven 将简单的停滞处理并抛出错误（无法找到依赖的文件）。
  • 步骤 4 － 在一个或多个远程仓库中搜索依赖的文件，如果找到则下载到本地仓库已被将来引用，否则 Maven 将停止处理并抛出错误（无法找到依赖的文件）。

3 使用maven进行项目周期的管理

3.1 创建project

Maven 使用原型 archetype 插件创建项目。参数说明:

  • DgourpId: 组织名，公司网址的反写 + 项目名称
  • DartifactId: 项目名-模块名
  • DarchetypeArtifactId: 指定 ArchetypeId，maven-archetype-quickstart，创建一个简单的 Java 应用
  • DinteractiveMode: 是否使用交互模式

3.2 编译project

  • mvn clean: 清除编译后的文件。该命令执行完成之后，mvn项目目录下的target目录就会被删除。
  • mvn compile： 编译，只编译main目录下的代码，不会编译test目录中的代码。
  • mvn test-compile命令：编译test目录下的代码。
  • mvn test: 运行test中的代码。
  • mvn package: 打包的命令，如果是java项目就会打成jar包，如果是web项目就会打成war包。其中jar包和war包的命名格式是按照pom.xml文件中的下面的部分得到的。
  • mvn install:发布项目到本地仓库，常用在jar包，将jar包放在本地仓库。
  • mvn tomcat：run : 一键构建项目
