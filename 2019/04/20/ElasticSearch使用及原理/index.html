ElasticSearch使用及原理

Posted on 2019-04-20 Edited on 2020-02-22 In 后台开发

1 入门基础

1.1 安装部署

ElasticSearch安装：
在物理机上部署：官网下载es，然后直接解压，执行bin/elasticsearch，启动不能使用root用户，通过指定-d参数可以实现后台启动。
也可以采用docker进行部署，参考地址

安装插件: bin/elasticsearch-plugin install 插件名字;

1.2 基本概念

es中存储的是文档，这些文档是使用json进行序列化的。

  • 文档可以理解为关系型数据库中的行记录，
  • es中的索引可以理解为关系型数据库中的数据库，每个index都有自己的mapping定义，用于定义包含文档的字段名和字段类型，
  • 类型可以理解为关系型数据库中的表，6.0 deprecated, 7.0废弃一个索引只能创建一个_doc

分片

ElasticSearch将数据分散到多个Lucene索引上，这些Lucene称为分片。数据高可用的问题是通过副本实现的。副本是主分片的拷贝。

倒排索引

词对文档ID的映射。Elasticsearch为了能快速找到某个词，将所有的词排个序，二分法查找term，logN的查找效率，就像通过字典查找一样。ElasticSearch为了提高性能，直接通过内存查找词

集群架构

集群中的节点有两个角色：管理节点、数据节点。ES将数据分散到多个分片中，分片可以分为：主分片和副本分片两种。

在创建索引时：客户端将请求发送到集群中的某个节点，如果该节点没有对应的主分片或者只有副本，那么这个请求会被转发到拥有主分片的节点上。

2 查询

ES提供了一个基于rest的API，通过API提交的数据有URL携带或以json文档的形式有HTTP消息体携带。一般来说，查询分为两个阶段：分散阶段、合并阶段。

API查询

  • 创建索引: curl -X PUT "localhost:9200/索引名字"

  • 删除索引：curl -X DELETE 'localhost:9200/索引的名字'

  • 查看es中的所有索引: curl -XGET 'localhost:9200/_cat/indices?v&pretty'

  • 插入文档

    上述api中: accounts表示索引的名字，person表示类型的名字，1表示带插入的文档的id。也可以不指定id但是要使用post发送请求，如下

  • 查询指定索引中的指定类型的某个id的文档数据

  • 查询指定索引的指定类型的所有的文档数据

  • 删除指定id的文档

  • 更新文档

DSL查询

DSL 查询主要包含两种类型的查询语句：

  • 叶子查询语句 ： 用于查询特殊字段的特殊值，例如：match, term, range 等
  • 复合查询语句 ： 使用bool可以合并其他的叶子查询或复合查询，从而实现非常复杂的查询逻辑

设置显示字段：_source，fields
查询语句：query
排序语句：sort

3 原理部分

3.1 分布式架构

每个index都会创建一些分片，这些分片分散在集群中不同的节点上，同时每个分片可能会创建适量的副本。在进行配置的时候应该选择合适的分片及副本数量。

在写入数据的时候，写入请求会先根据_routing规则确定将请求发送给哪个shard，请求接着会发送给Primary Shard，在Primary Shard上执行成功后，再从Primary Shard上将请求同时发送给多个Replica Shard，请求在多个Replica Shard上执行成功并返回给Primary Shard后，写入请求执行成功，返回结果给客户端。

写入请求到达Shard后，先写Lucene文件，创建好索引，此时索引还在内存里面，接着去写TransLog，写完TransLog后，刷新TransLog数据到磁盘上，写磁盘成功后，请求返回给用户。
